CSS Variable Scope Explained

Key Concept: Variables defined in :root are global. Variables defined in a specific element are only available to that element and its children.

Box 1 - Local Scope

This box defines --local-color: red

The variable is only available inside this box and its children.

✓ I can use --local-color here!

Box 2 - Different Local Scope

This box also defines --local-color: green

Same variable name, but different value and scope!

✓ I can use my own --local-color here!

Box 3 - Out of Scope

✗ This box CANNOT use --local-color from Box 1 or Box 2

✓ But it CAN use --global-color and --global-spacing

Why? Because global variables are available everywhere!

Box 4 - Overriding Global

This box redefines --global-color locally

Inside this box, --global-color is orange (not blue)

Outside this box, --global-color is still blue

This is called "overriding" or "shadowing"

Summary:

  • Global (:root): Available everywhere in the document
  • Local (specific element): Only available in that element and its children
  • Override: You can redefine a global variable locally for a specific section
  • Inheritance: Child elements can access their parent's variables

Think of it like this:

Global variables are like public announcements - everyone can hear them.

Local variables are like conversations in a room - only people in that room (and smaller rooms inside it) can hear them.
